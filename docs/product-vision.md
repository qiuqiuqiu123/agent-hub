# Agent-Hub 产品理念与设计方案
---

## 1. 产品愿景

**一句话：让中小公司的重复性业务流程，像写配置文件一样简单地自动化。**

展开来说：每家公司都有大量"人肉跑流程"的工作——定时汇总数据、审核代码后通知团队、收集反馈后分类归档、定期产出内容并发布。这些工作规则明确、步骤固定，但因为涉及多个系统（飞书、GitHub、邮件、公众号）的串联，以及部分环节需要"理解"和"创造"（比如写摘要、生成图片），传统自动化工具要么搞不定，要么太贵太重。

Agent-Hub 的答案是：**把业务 SOP 翻译成 Pipeline，由 AI Agent 处理"需要动脑"的环节，由 Tool Agent 处理"确定性调用"的环节，全自动执行、全自动输出。**

我们不是在做一个通用 AI 聊天工具，而是在做一个**业务流程自动化引擎**——AI 是其中的一个零件，不是全部。

---

## 2. 解决什么问题

### 痛点一：重复劳动吞噬生产力

中小公司的程序员和产品经理，每天有 30-50% 的时间花在"跑流程"上：

- 每天早上手动汇总昨天的 git 提交，写日报发飞书
- 每次有客户反馈，手动分类、创建 issue、通知相关人
- 每周写周报，翻遍各个系统拼凑数据
- 内容团队每天找热点、写文案、配图、发布，重复且耗时

这些工作不难，但**占时间、容易忘、质量不稳定**。

### 痛点二：现有工具的断层

| 工具类型 | 能做什么 | 做不到什么 |
|---------|---------|-----------|
| Zapier/n8n | API 对 API 的连接 | 无法处理需要"理解"的环节（写摘要、分类、创作） |
| ChatGPT/Claude | 理解和创造 | 无法自动触发、无法调用外部系统、无法编排多步骤 |
| Dify/Coze | AI 工作流 | 偏对话场景，业务集成弱，部署重 |

**Agent-Hub 填补的正是这个断层：既能编排确定性流程，又能在关键节点嵌入 AI 能力。**

### 痛点三：部署和成本门槛

大公司可以用 Airflow、Temporal 这类重型编排引擎。中小公司要的是：

- 一台服务器就能跑
- 不需要运维团队
- 按需付费（只有 AI 调用产生成本）
- 配置即上线，不需要写代码

## 3. 核心设计理念

### 3.1 Pipeline as Code

每个自动化流程是一个 Pipeline，用声明式配置定义：

```
Pipeline = 触发条件 + 步骤序列 + 输出目标
```

好处：
- **可版本管理**：Pipeline 配置可以用 Git 管理，可回滚
- **可复用**：一个 Pipeline 可以被复制、修改、分享
- **可组合**：步骤之间通过模板变量传递数据，像搭积木

### 3.2 Agent 分层：AI Agent + Tool Agent

这是 Agent-Hub 最核心的设计决策：

| | AI Agent | Tool Agent |
|--|---------|-----------|
| 驱动方式 | LLM（Claude/Codex） | 确定性程序调用 |
| 适用场景 | 需要理解、分析、创造 | API 集成、数据搬运 |
| 成本 | 按 token 计费 | 几乎为零 |
| 可靠性 | 概率性（需要 structuredOutput 约束） | 确定性 |
| 举例 | 写摘要、分类反馈、生成文案 | 发飞书消息、创建 issue、发邮件 |

**为什么要分层？**

1. **成本控制**：不是所有步骤都需要 AI，能用确定性调用解决的就不浪费 token
2. **可靠性**：Tool Agent 的行为 100% 可预测，减少整体流程的不确定性
3. **可调试**：出问题时能精确定位是 AI 环节还是集成环节

### 3.3 触发驱动

Pipeline 不需要人去"点击运行"，而是由事件驱动：

- **Cron 定时**：每天 8 点生成日报、每小时抓取热点
- **Webhook**：代码提交时触发审查、收到客户反馈时触发处理
- **手动/API**：需要时一键触发，或被其他系统调用

触发驱动意味着：**配置好就不用管了，流程自己跑。**

**数据流示例（热梗漫画场景）：**

```
Cron 每天 9:00 触发
  → AI Agent: 分析今日热点，选择适合做漫画的梗
  → AI Agent: 生成漫画文案和画面描述
  → Tool Agent (GPT Image): 根据描述生成漫画图片
  → Tool Agent (微信公众号): 组装图文并发布
```

## 5. 已验证场景：热梗漫画自动生成

这是我们第一个跑通的完整场景，已在公众号「夜猫子agent工坊」上线运行。

### 流程详解

| 步骤 | Agent 类型 | 具体动作 |
|------|-----------|---------|
| 1. 触发 | Cron | 每天定时触发 |
| 2. 热点分析 | AI Agent (Claude) | 分析当日互联网热点，筛选适合做漫画的话题 |
| 3. 文案创作 | AI Agent (Claude) | 生成漫画标题、对话文案、画面描述 |
| 4. 图片生成 | Tool Agent (GPT Image 2) | 根据画面描述生成漫画风格图片 |
| 5. 内容发布 | Tool Agent (微信公众号) | 组装图文素材，调用公众号 API 发布 |

### 验证了什么

- Pipeline 编排能力：5 个步骤顺序执行，上游输出作为下游输入
- AI + Tool 混合执行：AI 负责创意，Tool 负责执行
- 模板变量传递：每个步骤的 structuredOutput 被下游通过 dot-notation 引用
- 失败恢复：图片生成偶尔失败时，retry 策略自动重试
- 定时触发：Cron 每天准时执行，无需人工干预

### 效果

- 从"人工 2 小时/天"降到"全自动 0 人工"
- 内容质量稳定（AI 生成 + 人工偶尔审核）
- 发布频率从"想起来才发"变成"每天准时更新"

## 6. 商业化方向

### 方向一：SaaS 平台（主线）

- 免费层：3 个 Pipeline、基础触发、社区 Tool Provider
- Pro 层：无限 Pipeline、高级触发（Webhook）、优先执行队列
- Team 层：团队协作、权限管理、审计日志
- 定价锚点：$29-99/月（对标 Zapier 的 $49 起步，但 AI 能力更强）

### 方向二：私有部署

- 面向对数据安全有要求的公司
- 一键 Docker 部署（Next.js + SQLite，极轻量）
- 年费授权模式
- 适合金融、医疗、政府等行业

### 方向三：模板市场

- 用户可以分享/售卖自己的 Pipeline 模板
- 平台抽成模式（类似 Shopify 应用市场）
- 降低新用户上手门槛："选模板 → 填参数 → 上线"
- 形成网络效应：模板越多 → 用户越多 → 贡献者越多

### 收入模型

```
收入 = SaaS 订阅 + AI 调用加价（token 转售差价）+ 私有部署授权 + 模板市场抽成
```

其中 AI 调用加价是持续性收入：用户每次执行 Pipeline 中的 AI Agent 步骤，我们在 LLM API 成本上加 30-50% 的服务费。用户用得越多，收入越高。

## 7. 竞品对比

| 维度 | Agent-Hub | Zapier | n8n | Dify | Coze |
|------|-----------|--------|-----|------|------|
| **定位** | 业务流程自动化 + AI | API 连接器 | 开源自动化 | AI 应用开发 | AI Bot 平台 |
| **AI 能力** | 原生集成，作为流程节点 | 插件形式，能力有限 | 插件形式 | 核心能力 | 核心能力 |
| **业务集成** | 飞书/GitHub/邮件/公众号 | 5000+ 集成 | 400+ 集成 | 弱 | 弱 |
| **编排能力** | DAG + 条件 + 并行 + 恢复 | 线性为主 | DAG | 线性为主 | 线性为主 |
| **部署方式** | 单机轻量部署 | 纯 SaaS | 自托管/云 | 自托管/云 | 纯 SaaS |
| **目标用户** | 程序员 + 产品经理 | 非技术人员 | 技术人员 | AI 开发者 | 非技术人员 |
| **中国生态** | 原生支持（飞书/公众号） | 不支持 | 社区插件 | 部分支持 | 原生支持 |
| **成本** | 低（SQLite，无中间件） | 高（按任务计费） | 中 | 中 | 低 |

### 我们的差异化优势

1. **AI + 确定性执行的混合架构**：不是纯 AI 工具，也不是纯自动化工具，而是两者的最佳组合
2. **中国本土生态**：飞书、微信公众号等国内工具的原生支持
3. **极轻量部署**：一台 2C4G 服务器就能跑，不需要 Redis、Kafka、PostgreSQL
4. **Pipeline as Code**：对程序员友好，可版本管理、可复用、可分享
5. **成本优势**：只在 AI 环节产生 token 费用，Tool Agent 调用几乎零成本

- [x] Pipeline 引擎核心（顺序/并行/条件/变量传递）

- [x] Tool Provider：飞书、GitHub Issue、Email、GPT Image、微信公众号

## 总结

Agent-Hub 不是又一个 AI 聊天工具，也不是又一个 Zapier 克隆。它是两者的交叉点：

**用 Pipeline 编排业务流程，用 AI Agent 处理需要智能的环节，用 Tool Agent 处理确定性集成，用触发器让一切自动运行。**

技术上，我们选择了最轻量的栈（Next.js + SQLite），让一台服务器就能承载完整能力。商业上，我们瞄准中小公司最痛的"人肉跑流程"问题，用 SaaS + 模板市场的模式实现规模化。

MVP 已经验证了核心假设：Pipeline 引擎可以稳定运行，AI + Tool 混合执行是可行的，真实场景（热梗漫画）已经跑通。下一步是产品化和用户增长。
